    5年运维经验分享：一个小白走向高级运维工程师之路

    我是Freeman，88年的，老家河南，来上海4年，O2O行业高级运维工程师，拥有5年运维经验。

    我目前维护上千台服务器，熟悉大型网站架构，熟悉集群高可用，熟悉数据库。对大并发场景下的业务稳定性可用性有丰富经验，参与公司多个运维核心项目，不断改进和完善自动化平台及流程，关注NoSQL和分布式存储和大数据的发展。

    今天给大家讲讲我的运维学习故事

    在踏上运维路之前，我并不了解运维，但机缘巧合做了运维之后，对于运维的苦闷是颇有心得的，重复性的工作，牵一发而动全身的割接和上线，大量消耗精力和时间的工作事项是运维人最苦闷的地方。

    在刚刚进行运维工作的时候，有很多不懂的问题，虽然能在工作中与同事交流能获得好的方法，但总是杯水车薪，感觉太慢了。

    在工作中处处受制与人，为了做好这份工作，同时也为了更加自由，我渐渐明白了做好运维需要系统的专业而全面的知识，然而这些知识都不是一天两天能通过简单学习就能掌握的。

    为此我陷入了极度的苦恼与恐慌中，不得解脱。

    在面对工作中的巨大压力和困惑的时候，我领悟到了只有自强不息，才能使自己受人尊重，只有不断精进才能受人待见，也只有战胜自己才能获得自由展现自我价值。

    在对运维的工作进行了细致的分析之后，抱着精进学习的目的，我坦然的选择了一家培训机构，进行系统的学习。

    # 踏上征服Linux的征程

    因为和你们一样，甚至不如你们，我是几乎零基础参加的培训，只看过几页鸟哥Linux基础，来的时候键盘都敲不熟练。对的，我是行外人，学的自动化，当然不是如今炙手可热的运维自动化，而是偏电气方向的。

    这么说吧，就第一天讲Linux基础的时候，我跟上了，因为这些只要听听就行了从第二天起，就赶不上了，还好有了面授班这个氛围(现在网络班的交流平台也挺好的)，大家可以一起做实验，互相帮忙。

    我几乎是白天吃力的听课，课下练习，让其他同学帮我排错，晚上回去补。补网络知识，补当天的课，还向预习一下新课，睡得太晚，第二天没精神，注意力不集中更跟不上，这样是一个恶性循环，调整了几次时间，发现看到晚上一点多睡觉挺合适就这个状态下来的。

    当课程快结束的时候，老师在座位上向我们笑着挥手说再见，顿时我傻了。啊！结束了我还没来得及回味一下，就像看场精彩的电影，还没过瘾，就演完了。当时掌握的程度连30%不到吧。

    # 英雄，还是狗熊？

    当时也不敢去找工作，不自信，就在附近的大学自己看视频学了一段时间，留下来学习的有三四个。3个月刚刚熟悉就分开了，心情着实低落了几天。

    但生活还要继续的，附近有个小咖啡馆有暖气，往期的老学员应该知道吃饭会路过那里的，我们在那蹭过人家的wifi，由于老不喝咖啡，挺不好意思的。

    当时在做nagios时，服务老起不来，后面排查了很久，发现是iptables没关。当时就是那么菜，尽管现在也菜，呵呵看了一天原理没搞懂，实验不顺利，那天也不怎么好。

    对比之前轻松混日子的工作，半夜不由哭了起了来，大哭，现在还记忆犹新，哭了之后感觉格外的舒畅，后面好像睡得很香，很踏实。第二天，又去农大看nagios，当即拿下，原理透彻明了，实验顺利。

    对技术的第一次升华，应该就在这里。后面自信心莫名的大增，后面乘着这个余威，学习了负载均衡高可用集群，MySQL备份复制，主从等。

    后面在郑州找了个小公司运维岗位的工作做着，算是一个锻炼吧，一次在内部群里聊天，上海和北京的小伙伴说运维环境多好，也就想去试试，就和几个小伙伴商量一起来大城市了。

    # 如何立足职场这个没有硝烟的战场？

    就像现在的年轻人有的北漂，有的南下，带着渴望但又不情愿的感觉来到一个大城市，仿佛不想上学的小学生去上学，走1步退2步的心情，这个不情愿呢，多是不自信，那种对未知的恐惧感。

    一起来的几个人呢，水平都不太高，面试刚开也不太顺利，不懂的回来讨论，找答案，问群里的师兄师姐，问老师，如斯者多次，大家都找到工作了，平均5，6k以上，挺高兴的，感觉比之前的收入高了不少。

    期间，老师和往期的小伙伴们提供了很多笔试题，建议和指导，谁的公司招人，这个公司谁在里边待过等等对我们帮助很大，真的感谢这帮充满活力和互帮互助的小伙伴们，感觉我们不再孤单。

    到新公司上班，位于一个环境优雅的写字楼里，心里既兴奋又不安，大城市的运维就是正规，终于从游击队变成正规军了。前一周，经理让我看运维培训手册，心里踏实了，就TM这样啊，大部分学过，而且比这系统详细。开始上马，中间难免遇到各种小问题，小困难，那就网上搜，问往期的学员们。

    后面我搬到公司的附近住，刚好离大学也很近，后面的成长很多还是和大学有很大的关系，平时在公司义务加班两个小时，回来有时看一到两个小时视频。周末我背着电脑，在这里看视频，当时学生们在准备考研，学习气氛很浓，还要和他们抢位置，不管那么多了，和他们一起学。

    # 职场如何领先一步？

    就这样坚持下来了，到了13年底，视频普遍看了两三遍以上，实验也跟着做，核心的web，MySQL，集群高可用等看了更多遍，不断总结笔记+实验。后面公司不是不忙，就酝酿着该换个工作了，就在网上搜了些面试，礼仪相关的资料，开始全面的准备工作，其实我一直在准备着。

    年后开始上班时我并没有一下就去找，等待着年后招聘旺季，不是有金三银四的说法嘛，所以，又温习了下之前的笔记。

    # 蜕变，还是掉坑？

    不多久，一家O2O公司联系到了我，电话面试后，才去公司面试，先后有三四位技术官面试，每个人侧重点不一样，有原理，有实战，有细节，有深入，涉及web，数据库，监控，负载均衡类型，高可用，大规模网站架构，调优等等，有的直接给出一个需求让我给出解决方案。

    我去，大公司就是会玩啊，用到了这么多东西顿时越发兴奋了，当时觉得这就是我想要的公司。

    然后和人资HR谈待遇，没想到下午很快就接到了电话，而且待遇还经过了讨价还价，感觉和菜市场差不多的赶脚，可以商量的。待遇还可以，有一家小公司，工资有和这差不多的，因为这个平台的缘故，听同事说O2O模式不错，又接受了新投资，在一个快速发展期。

    记得老师和我们说过，生活就是一场折中和妥协。这是一个大智慧，希望我们都慢慢去体会，学会应用。

    # 致谢

    像我零基础，计算机边缘专业的同学，之前也没有编程思维，仅有的c语言也是睡过去的。我想说的是，我都可以坚持下来，有所突破，被认可，我有理由相信，你们会做的更好，走的更远。

    我们既没有到绝望的地方，也没有接触死亡的边缘，just just去学，如果有这样一个设想：你能学会，直接给你100万，学不会拉出去枪毙，我估计你们学的会更好。

    一定要有自驱动，自我驱动能力，技术到了一定层次，要学会独立研究，因为这时不是别人不想帮你，而是别人不一定有你知道的多呢。

    # 共勉

    以上是我的感受，坚持下来，一切都没有想象中的那么困难。回头看时，有苦涩，有泪水，也有汗水，有收获有喜悦，也有成长，更有那越来越多的自信心。任何一个蜕变都是痛苦的。

    破茧成蝶的确很美好，但那嫩嫩的肉质突破硬硬的皮层时，那苦楚的过程直到变成蝴蝶，你还觉得美好吗，当然你突破不了，你根本不知道翩翩蝴蝶给大千世界带来的那些美与享受。

    生活会给你最终的回报，就看你有没有资格看的到，如果你止步于破茧的那一刻，承受不了那一段的煎熬…你懂的。

    当然，祝贺你坚持了这么久，我要给你们肯定和鼓励，我感觉你们才刚刚开始，我既不能夸奖大家，因为路还很长；我也不能说别的，因为路确实还很长。

    仅是一些鼓励，说声加油！
